🔧 Prompting Techniques Covered

Technique 1 — Fill in the Blank
The process of deriving insights from data is called ...
💡
Leaving the sentence incomplete with ... or ? signals GPT to complete the thought rather than give a long explanation.
Technique 2 — One-Word / Constrained Answer
Using the example complete the request: Q: What is the most popular language for data analysis? A: Python Q: What is the most popular data file format for data analysis? A: ?
💡
Providing an example Q&A pair (few-shot prompting) teaches GPT the exact format you want — no extra explanation needed.
Technique 3 — Persona Prompting
You are Shakespeare. What is data analysis?

GPT answers in Shakespeare's style but still gives the correct answer. Use for presentations to non-technical stakeholders — give GPT the stakeholder's persona (e.g., "You are a CEO of an e-commerce shoe company") and ask it to explain your topic from their perspective.

Technique 4 — Let GPT Ask Questions First
I want a high-level overview of data analysis. I haven't done this before and don't know the terminology. Ask me as many questions as you need, then give me a relevant answer.

GPT will first ask about your background, data sources, tools, challenges — then tailor its explanation perfectly to you.

Technique 5 — Explain Like I'm 10
What is data analysis? Explain like to a 10-year-old.

Perfect for creating presentations for mixed audiences (sales, marketing, tech, management) — everyone can follow the explanation regardless of technical background.

For personal projects, learning, and internal tools at work — the free tier is perfectly sufficient. Pay only when you need production-grade API automation.

📊 Case Study 1: Categorical Encoding (Survey Data)

1

Establish Persona & Share Data Snippet

Start with a role prompt and paste a chunk of your CSV (column names + first few rows). This is critical — without data context, GPT's answers are generic.

Prompt Structure
You are a professional data analyst with 20 years of experience in cutting-edge technology. I have a data file called [filename] with responses to a survey on GenAI... Here is a chunk of that file: [paste column names + first 2–3 rows of actual data] After reading this, show me the list of all categorical variables from the dataset.
2

Remove Irrelevant Columns

Ask GPT to remove columns like respondent_id or IP address that add no analytical value.

3

Get Unique Values for Each Categorical Column

Ask GPT to generate a Python script that outputs each categorical column's unique values. Run it in Google Colab, then paste the output back to GPT.

Prompt
Create a Python script that will output all these variables with the list of unique values in a format that I can easily copy-paste back here.
4

Build Encoding Table

Ask GPT to create a two-column table: Variable Name | Suggested Encoding Method. Review it — GPT often makes mistakes here!

⚠️
GPT is not always right on encoding! Use your analyst knowledge to verify. For example, "prefer not to say" should get a negative/lowest weight, not the highest ordinal value.
5

Correct GPT's Mistakes Explicitly

Correction Prompt
Update the table with the following changes: - Consider age as ordinal column - Consider gender as nominal column - For prefer not to say and others, assign negative value (e.g. -1) as they should have lowest weight
6

Add Encoded Value Examples Column

Add a third column showing actual old → new value mappings (e.g., "18–24" → 1, "25–35" → 2). Use few-shot examples from earlier columns to enforce the format.

7

Convert All to JSON

Prompt
Convert the above table to JSON format, leaving out the encoding method column. Output format example: [paste format example]
8

Merge JSON Files & Apply Encoding to CSV

GPT writes a Python script to merge the JSON encoding files, then applies them to the original CSV — adding new columnname_enc columns right after their source column (in order).

🛒 Case Study 2: E-Commerce Churn Prediction

🎯
Dataset: Electronics store event logs (view, cart, purchase events). Challenge: No explicit churn column — GPT helps define what "churn" means!
Column Description Notes
event_time When the event happened Used to compute recency
event_type view / cart / purchase / remove_from_cart Core signal for engagement
product_id Product identifier Many-to-many with user_id
user_id Customer identifier Used for churn definition
user_session Browser session key Can contain multiple events

Churn Definition (GPT-suggested)

GPT's Suggestion (Accepted)
A user is considered churned if they have not made a purchase within the last 30 days relative to the maximum date in the dataset. Instructor note: GPT takes the max date as reference (not each user's last date) for statistical consistency — ask GPT to explain its reasoning if confused!

Feature Engineering via GPT

🔮

GPT-Proposed Features (Round 1)

Recency of activity, frequency of activity, total purchases, unique products viewed, average session duration, cart abandonment rate, and more — 10 features at once.

Filter step: After getting probabilities, tell GPT: "Only keep features where confidence interval is high or probability of success is above 70%." GPT will filter the table automatically.

🧠 Handling GPT's Memory Problem

⚠️
GPT has short-term memory (like Ghajini!) In long conversations, it forgets earlier context. When this happens, re-establish context in your next prompt.
Memory-Refresh Prompt Pattern
Let me repeat a few things for you because of memory constraints. I have a dataset [filename]. [Paste dataset description again] Here are the first few lines: [paste data chunk again] As part of our previous discussion, you already gave me the following ideas: [paste the output from the previous step] Now, [new question or next step]…

Best Practices

Golden Rules for Using GPT

✅

Be Explicit & Specific

Always name your file/folder, describe the columns, and state the exact output format you want. Vague prompts → vague results.

🔁

Paste Errors Back

When code throws an error, copy the full error message, paste it back to GPT, and say "Rewrite the script." GPT fixes most errors on the first try.

📋

Give Sample Data

For data analysis tasks, always paste 2–3 rows of your actual data. This dramatically improves the accuracy of GPT's code and insights.

🎭

Use Few-Shot Prompting

Provide one or two examples of the output format you want. GPT will match it exactly — this is the most reliable way to control output format.

🏷️

Use Delimiters

Use ### or --- to separate sections of your prompt (context from question). Helps GPT understand where the data ends and your request begins.

🧠

Verify GPT's Output

GPT makes mistakes — especially in encoding type decisions and statistical reasoning. Use your domain knowledge to verify every suggestion before applying it.

🚀 The Big Picture

GPT is not replacing you — it's making you 2–3× faster. A project that used to take a full day can now take 4–5 hours. The analysts who know how to use GPT well will outcompete those who don't. Learn the fundamentals, use GPT as your accelerator.
